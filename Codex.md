# Codex.md - 协作约定

本文件补充 `CLAUDE.md`，供 Codex 在 EssayPad 项目中遵循；如有冲突，以用户的最新指令为准。

## 沟通

- 使用中文，表达简洁直接；用户是后端开发，后端改动可以说明关键协议和存储影响。
- 用户已明确要求可直接修改时，先在现有代码中确认边界后直接实现，不重复索要批准。
- 不回退或覆盖用户已有改动；改动笔记、日记等相邻功能时，保持原功能独立可用。

## 实现

- 优先复用项目现有模型、接口和 UI 模式，不为局部需求引入不必要的依赖、表字段或接口字段。
- 新功能先补能验证核心行为的测试；完成后执行与改动匹配的构建和测试。
- macOS 工程中的新增 Swift 文件必须同时加入 Xcode target，不能只存在于磁盘。

## 提交

- 每次完成一次用户触发的修改，且构建与验证通过后，必须创建一个 Git commit。
- commit 信息使用中文，格式为 `type(scope): 说明`；同一逻辑单元的改动合并为一个 commit。
- 不提交半成品；若验证未通过，说明实际阻塞原因并继续处理。
