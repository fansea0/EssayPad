# 日记功能设计

## 1. 功能目标

新增独立「日记」功能，入口与「笔记 / 任务」同级。

日记不放入现有笔记分类中，不复用 `bug / 需求 / 想法 / 草稿` 这套分类心智。日记按日期组织，每天最多一篇，正文编辑复用当前 Markdown 编辑器能力。

核心目标：

- 独立入口：主功能切换增加「日记」。
- 独立存储：后端新增 `diary_entries` 表。
- 复用编辑器：正文继续使用现有 Markdown 编辑器。
- 结构化记录：新增 `心情 / 状态 / 活动` 三个字段。

## 2. 产品形态

主入口：

```text
笔记 / 任务 / 日记
```

日记页结构：

```text
左侧：
- 搜索日记
- 全部 / 本周
- 日期列表
- 列表展示：标题 + 摘要 + 心情 · 状态 · 活动

右侧：
- 最近 7 天日期条
- 日记标题
- 心情 / 状态 / 活动 三组轻量选择
- Markdown 编辑区，复用当前编辑器
- 自动保存状态
```

默认规则：

- 进入日记页默认打开今天。
- 今天没有日记时，展示空编辑态。
- 用户输入标题、正文或选择任一结构化字段后创建日记。
- 每个自然日最多一篇日记。
- 删除走软删。

## 4. 数据库设计

当前项目后端使用 SQLite，本地实现先使用 SQLite DDL。字段设计保持可迁移到 MySQL。

### 4.1 SQLite DDL

```sql
CREATE TABLE IF NOT EXISTS diary_entries (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    user_id BIGINT NOT NULL DEFAULT 0,
    diary_date BIGINT NOT NULL DEFAULT 0,
    title VARCHAR(200) NOT NULL DEFAULT '',
    content TEXT NOT NULL DEFAULT '',
    mood TINYINT NOT NULL DEFAULT 0,
    status TINYINT NOT NULL DEFAULT 0,
    activity TINYINT NOT NULL DEFAULT 0,
    created_at BIGINT NOT NULL DEFAULT 0,
    updated_at BIGINT NOT NULL DEFAULT 0,
    is_deleted TINYINT NOT NULL DEFAULT 0
);

CREATE UNIQUE INDEX IF NOT EXISTS uniq_diary_user_date
ON diary_entries(user_id, diary_date);

CREATE INDEX IF NOT EXISTS idx_diary_user_updated
ON diary_entries(user_id, updated_at DESC);

CREATE INDEX IF NOT EXISTS idx_diary_user_date_deleted
ON diary_entries(user_id, diary_date DESC, is_deleted);
```

### 4.2 MySQL DDL

```sql
CREATE TABLE `diary_entries` (
  `id` int(11) NOT NULL AUTO_INCREMENT COMMENT '自增ID',
  `user_id` bigint(20) NOT NULL DEFAULT 0 COMMENT '用户id,单机版默认0',
  `diary_date` BIGINT(11) NOT NULL DEFAULT 0 COMMENT '日记日期0点时间戳',
  `title` VARCHAR(200) NOT NULL DEFAULT '' COMMENT '日记标题',
  `content` text NOT NULL COMMENT 'Markdown正文',
  `mood` TINYINT(1) NOT NULL DEFAULT 0 COMMENT '心情:0未选择/1开心/2平静/3低落/4焦虑',
  `status` TINYINT(1) NOT NULL DEFAULT 0 COMMENT '状态:0未选择/1很好/2较好/3普通/4较差/5很差',
  `activity` TINYINT(1) NOT NULL DEFAULT 0 COMMENT '活动:0未选择/1工作/2学习/3出游/4休息/5游戏',
  `created_at` BIGINT(11) NOT NULL DEFAULT 0 COMMENT '创建时间戳',
  `updated_at` BIGINT(11) NOT NULL DEFAULT 0 COMMENT '更新时间戳',
  `is_deleted` TINYINT(1) NOT NULL DEFAULT 0 COMMENT '0未删除/1已删除',
  PRIMARY KEY (`id`),
  UNIQUE KEY `uniq_diary_user_date` (`user_id`, `diary_date`),
  KEY `idx_diary_user_updated` (`user_id`, `updated_at`),
  KEY `idx_diary_user_date_deleted` (`user_id`, `diary_date`, `is_deleted`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_general_ci COMMENT='日记表';
```

  "activity": 1

## 6. 后端逻辑约束

### 6.1 日期规则

日记日期统一使用本地时区当天 0 点时间戳。

后端列表中 `week` 范围建议按当前本地自然周或最近 7 天实现。为了和当前原型一致，优先使用最近 7 天。

### 6.2 每天一篇

通过唯一索引保证：

```sql
UNIQUE(user_id, diary_date)
```

`POST /diaries` 使用 create-or-update 语义：

- 不存在：创建。
- 已存在：更新。

### 6.3 无变化不更新

更新时如果 `title/content/mood/status/activity` 均无变化，不刷新 `updated_at`。

这样避免用户只是打开日记，列表顺序或更新时间被动变化。

### 6.4 枚举校验

服务层校验枚举范围：

```text
mood: 0-4
status: 0-5
activity: 0-5
```

非法值建议返回 400。

## 8. 自动保存设计

### 8.1 触发条件

以下字段变化时触发：

```text
title
content
mood
status
activity
```

### 8.2 Debounce

延迟 600ms 保存，参考 `NoteEditorView`。

### 8.3 创建条件

当日记尚不存在时，满足以下任一条件才创建：

- 标题不为空
- 正文不为空
- mood/status/activity 任一字段非 0

### 8.4 离开页面

切换日期或离开日记页时：

- 取消 pending autoSave task。
- 立即保存一次。
- 如果仍为空草稿，不创建。

## 9. 空态设计

今天无日记：

```text
标题为空
正文为空
心情/状态/活动未选择
展示编辑态
```

历史日期无日记：

```text
展示“这一天还没有记录”
提供“补写这天”按钮
```

## 12. 风险与注意事项

- 日记每天唯一，必须依赖唯一索引兜底，不能只靠前端判断。
- 自动保存必须避免 no-op update，否则会造成 `updated_at` 抖动。
- 日期必须统一成本地 0 点时间戳，否则跨天和 week 查询会错。
- Markdown 图片仍使用现有 `images/...` 机制，不为日记单独做图片存储。
- 日记表独立后，AI 周报是否纳入日记内容后续再设计，本期先不混入。
