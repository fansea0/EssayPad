# Flutter 移动端 MVP 设计

## 目标

在 `mobile/` 初始化 EssayPad 的 Flutter iOS/Android 客户端，提供低阅读负担首页、紧凑并排的笔记与日记入口，以及可离线保存的基础笔记编写流程。

## 页面

- 首页：日期、笔记与日记并排入口、最近笔记列表、底部导航。
- 笔记：列表展示、本地新建、编辑、删除。
- 编辑器：标题、分类、Markdown 正文和常用 Markdown 快捷操作。
- 任务、周复盘、我的：首版提供可导航的占位页面，避免导航结构推倒重来。

## 数据与边界

首版不连接现有 Go 服务，因为该服务仅监听桌面本机 `127.0.0.1`。笔记通过 `shared_preferences` 本地持久化；`NotesStore` 是唯一的数据入口，未来可在不影响页面的前提下替换为 SQLite 与远端同步实现。

API Key、AI 配置和同步都不进入本次移动端 MVP。

## 视觉原则

不使用大面积阅读卡片。首页内容位于一个连续的浅色背景中，笔记和日记入口尺寸相同、相邻排列；正文密度适合手机单手浏览。颜色使用深墨、青绿、珊瑚橙和柔和灰白，圆角不超过 8pt。

## 验证

- Flutter analyzer 无错误。
- 单元测试覆盖本地笔记新增、更新、删除。
- Widget 测试覆盖首页并排入口和新建笔记入口。
